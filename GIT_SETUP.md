# Настройка Git для проекта

## Быстрая настройка

Репозиторий уже инициализирован. Для применения всех настроек выполните:

```bash
# Применить локальные настройки Git (опционально)
git config --local --file .gitconfig.local

# Или вручную настроить шаблон коммита
git config commit.template .gitmessage
```

## Структура файлов Git

- **`.gitignore`** - файлы и папки, которые Git будет игнорировать
- **`.gitmessage`** - шаблон сообщения коммита
- **`.gitconfig.local`** - локальные настройки Git (алиасы, цвета и т.д.)
- **`CONTRIBUTING.md`** - подробные правила коммитов

## Первый коммит

```bash
# Добавить все файлы
git add .

# Создать первый коммит
git commit

# Или с коротким сообщением
git commit -m "chore: начальная настройка проекта"
```

## Использование шаблона коммита

При выполнении `git commit` без флага `-m` автоматически откроется редактор с шаблоном из `.gitmessage`.

Для принудительного использования шаблона:
```bash
git commit -t .gitmessage
```

## Полезные алиасы

После применения `.gitconfig.local` доступны следующие алиасы:

```bash
git st          # статус
git lg          # красивый лог
git ci          # коммит
git br          # список веток
git co          # checkout
git df          # diff
git unstage     # отменить добавление файла
```

## Правила коммитов

См. подробные правила в файле `CONTRIBUTING.md`.

### Краткая справка:

**Формат:** `<type>(<scope>): <subject>`

**Типы:**
- `feat` - новая функциональность
- `fix` - исправление ошибки
- `docs` - документация
- `refactor` - рефакторинг
- `test` - тесты
- `chore` - настройки, задачи

**Примеры:**
```bash
git commit -m "feat(app): добавлена загрузка переменных из workspace"
git commit -m "fix(graph): исправлена синхронизация таблицы и графика"
git commit -m "docs: обновлена документация по API"
```

## Проверка настроек

```bash
# Проверить текущие настройки
git config --list --local

# Проверить шаблон коммита
git config commit.template
```
